🍼 BIODOSE: THE EASY-PEASY WORKFLOW & LIBRARY GUIDE 🍼
--------------------------------------------------------------------------------
This guide is written in extremely simple, conversational, and analogy-driven language 
so that anyone (even without a heavy tech background) can grasp exactly what BioDose 
does, how the data flows, and what every single library does.

================================================================================
🌟 THE CORE IDEA: WHAT IS BIODOSE?
================================================================================
Most fitness apps are like "Calorie Counters." They only count the big blocks (Carbs, 
Protein, Fat). 
BioDose is different. It is a "Chemical & Safety Guardian." It looks at the tiny, hidden 
ingredients—like food colorings, chemical preservatives, and processing agents. It then 
checks if these tiny ingredients are safe or dangerous specifically for YOUR body 
(based on 21 different health conditions, like diabetes, asthma, or peanut allergies).

================================================================================
🔄 STEP-BY-STEP WORKFLOW: HOW DATA FLOWS
================================================================================

1. 📸 THE SCAN (Getting the Barcode)
   - You point your phone camera at a bag of chips.
   - The app's camera detects and crops the black-and-white lines (barcode).

2. 🌐 THE DATABASE QUERY (What is this product?)
   - The app takes the barcode numbers and sends them to OpenFoodFacts (a giant online 
     encyclopedia of all foods in the world).
   - OpenFoodFacts sends back the list of ingredients (e.g., "Water, Citric Acid, E120...").

3. 🧪 THE EXTRACTOR & MATCHER (Cleaning up the words)
   - Sometimes ingredients have typos or are written slightly differently.
   - The "Extractor" finds all the chemical codes (like E-numbers) and chemical names.
   - The "Matcher" acts like a spelling-corrector, matching the text to a local database 
     of toxic chemicals even if there are typos.

4. 🧮 THE SYNERGISTIC MATH SCORER (Calculating the danger)
   - It reads your locked medical profile (your health conditions represented by secret 
     codes like MDC01, MDC02, etc.).
   - It calculates a score from 0.0 (totally safe) to 1.0 (extremely dangerous) using 
     advanced medical rules:
       - Multi-Hit Rule: Having 3 bad chemicals that attack the same organ is much worse 
         than having just 1.
       - Severity Weight: A peanut allergy (instant danger) is weighed much heavier than 
         mild asthma.
       - Cross-Talk: If multiple health conditions are active, the app calculates how they 
         make each other worse.

5. 💬 MEDSENSEI AI CHATBOT (The friendly doctor)
   - An AI chatbot powered by Groq Cloud (Llama-3.1) reads the product details and your 
     medical profile.
   - It talks to you in friendly language, answering questions like: "Will this chips packet 
     give me an asthma attack?"
   - If the internet goes down, it falls back to a built-in offline rules engine so it 
     never stops working.

6. ⚙️ THE SHADOW MODEL (The silent student)
   - Every time a user scans a product, the transaction is logged.
   - In the background, a second AI model (the "Shadow Model") is secretly studying these 
     scans.
   - Every 1,000 scans, it retrains itself to become smarter, comparing its guesses against 
     the official math score.

7. 🔒 THE SECURITY LOCK (Don't let pranks happen!)
   - If someone tries to change your allergen settings (e.g., turning off your peanut allergy 
     toggle), the app locks up and asks for your account password to verify it is really you.

🛠️ THE TECH STACK & LIBRARIES (Explained with simple analogies)
================================================================================

--------------------------------------------------------------------------------
🔌 BACKEND (Python / Django) - The Brain of the System
--------------------------------------------------------------------------------

* Django (6.0.5) & djangorestframework (3.17.1)
  👉 Analogy: The skeleton and neurological system of our backend. It routes web traffic, 
     manages database access, and handles incoming phone requests securely.
* djangorestframework-simplejwt (5.3.1)
  👉 Analogy: The digital security pass. It issues temporary tokens so users don't have to 
     type their password on every single click.
* django-cors-headers (4.9.0)
  👉 Analogy: The gatekeeper. It allows the frontend website to talk safely to our backend 
     server without browser security blocks.
* django-redis (5.4.0)
  👉 Analogy: The cheat-sheet. It remembers frequent queries so the server doesn't have to 
     re-calculate the same barcode math over and over.
* python-dotenv (1.2.2)
  👉 Analogy: The locker key. It hides secret API keys and database passwords in a secure 
     environment file away from public eyes.
* gunicorn (26.0.0) & gevent (26.4.0)
  👉 Analogy: The crowd controllers. They allow our server to handle thousands of users 
     simultaneously without crashing.
* whitenoise (6.6.0)
  👉 Analogy: The fast-delivery counter. It serves images and CSS files directly to users 
     at lightning speed.
* psycopg2-binary (2.9.12)
  👉 Analogy: The translator. It translates Python requests into PostgreSQL database language.

--- Barcode & Vision Libraries (The Eyes) ---
* opencv-contrib-python
  👉 Analogy: The digital camera lens. It processes raw pictures, adjusts brightness, and 
     prepares barcode frames.
* pyzbar (0.1.9) & zxing-cpp (3.0.0)
  👉 Analogy: The barcode readers. They decode the black-and-white barcode lines into numbers.

--- Text Matching & NLP Libraries (The Spell-Checkers) ---
* RapidFuzz (3.14.5) & Levenshtein (0.27.3)
  👉 Analogy: The typo-tolerant dictionaries. If a scanned package writes "sulfit" instead 
     of "sulfite," they know it's the same thing and map it correctly.
* beautifulsoup4 (4.14.3)
  👉 Analogy: The webpage scraper. It helps clean up messy text from web pages.
* nltk (3.9.4) & regex (2026.4.4)
  👉 Analogy: The grammar teachers. They clean up raw sentences, split text into words, 
     and extract exact chemical symbols.

--- Machine Learning & Math Libraries (The Calculators) ---
* scikit-learn (1.8.0)
  👉 Analogy: The AI toolkit. Used for building the shadow model that predicts risk trends.
* numpy (2.4.4), pandas (3.0.2), scipy (1.17.1)
  👉 Analogy: The mathematical spreadsheets. They hold tables of chemical hazards and run 
     fast matrix calculations.
* shap (0.51.0) & interpret (0.7.8)
  👉 Analogy: The lie detectors. They explain exactly why an AI model made a certain decision 
     so humans can understand it.
* matplotlib (3.10.9) & plotly (6.7.0)
  👉 Analogy: The art class. They draw graphs, charts, and diagrams of user health trends.

--- Utilities ---
* requests (2.33.1) & httpx (0.28.1)
  👉 Analogy: The mail delivery birds. They send messages to the external Groq AI and 
     OpenFoodFacts servers.
* loguru (0.7.3)
  👉 Analogy: The flight recorder. It writes down everything the app is doing in a log file 
     so we can fix bugs if it crashes.
* pytest (9.0.3)
  👉 Analogy: The inspection officers. Runs tests on the code to make sure updates don't 
     break existing features.

🏥 MEDICAL PROFILE CODES: WHAT ARE THEY?
================================================================================
To protect privacy and keep data clean, health conditions are stored as codes:
- MDC01: Diabetes Type 2
- MDC02: Hypertension
- MDC03: Asthma
- MDC04: Celiac Disease
- MDC05: IBS (Irritable Bowel Syndrome)
- MDC06: Chronic Kidney Disease
- MDC07: Liver Disease
- MDC08: Thyroid Disorders
- MDC09: Autoimmune Conditions
- MDC10: ADHD
- MDC11: Heart Disease
- MDC12: Pregnancy
- MDC13: Lactation
- MDC14: Infants (0-2 years)
- MDC15: Children (3-12 years)
- MDC16: Elderly (60+)
- MDC17: Peanut Allergy
- MDC18: Shellfish Allergy
- MDC19: Dairy Allergy
- MDC20: Gluten Sensitivity
- MDC21: Soy Allergy
